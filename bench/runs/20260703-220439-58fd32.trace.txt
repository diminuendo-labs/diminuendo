To implement the `range_summary` function, I first handle the edge case where the input list is empty, returning an empty string immediately. Then, I sort and deduplicate the input list using `sorted(set(nums))`. The core logic involves iterating through the sorted unique numbers while keeping track of the start and end of consecutive sequences. When a break in the sequence is detected, either a single number or a range (start-end) is added to the result list, depending on whether the start equals the end. Finally, I join all elements in the result list with commas and return the string. This approach ensures that the function is both concise and easy to understand, adhering to the criteria provided.
