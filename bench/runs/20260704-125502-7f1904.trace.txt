I created a function `token_case(s)` that takes an underscore-delimited identifier as input. I first defined an exception list containing 'URL' and 'ID', which will keep their exact form wherever they appear in the identifier. The function then splits the input string by underscores, and processes each token using a list comprehension.

For every token that is not empty (meaning it exists in the string) and is not in the exception list, I lowercase the first letter and capitalize all subsequent letters (this is done by slicing `x[1:]`). For tokens in the exception list or any empty strings, I leave them as they are. Finally, I join the processed tokens together without a separator and return the result in title case. This approach handles the edge case where an identifier contains exceptions like 'URL' or 'ID'.
