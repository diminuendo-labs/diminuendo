I chose to implement the function using a HeapSort approach, which allows us to sort the k largest numbers efficiently. Initially, I check if `nums` is empty or `k` is less than or equal to 0 and return an empty list in those cases. If the length of `nums` is smaller than `k`, I sort it in descending order and return the first `k` elements. Otherwise, I create a list `heapsort` to store the k largest numbers, then iterate through each number in `nums`. I insert each number into its correct position in the sorted array using a binary search-like algorithm. In case of ties, Python's comparison operator will prioritize larger values at the end of the array, which satisfies the requirement for keeping duplicates. To handle the edge case where `k` exceeds the list length, I return all values sorted descending.
