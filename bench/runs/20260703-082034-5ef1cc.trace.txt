To solve this problem, I iterated through the string in reverse order to ensure that if multiple runs have the same length, the last such run's character is returned. I used a variable `current_char` to keep track of the current character being counted and `current_length` to count its consecutive occurrences. If the current character changes, I compare the `current_length` with the `max_length` found so far. If it's greater or equal, I update `max_length` and store the current character as the result. After the loop, I return the result based on which was longer or if there was a tie, the last occurring run's character is returned. This approach ensures that the function is both correct and efficient, with a time complexity of O(n) where n is the length of the string.
