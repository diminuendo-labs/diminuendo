I used a set to store distinct unordered pairs of numbers that sum up to the given target. Then I iterated over the list, for each number, I calculated its complement (the difference between the target and the current number). If the complement exists in the list, I created a pair with the smaller number as first element and the larger number as second element, added it to the set, and counted the number of distinct pairs.
The edge case I handled is when the same pair appears multiple times in the list; since sets don't allow duplicates, the function returns the correct count of distinct pairs.
