I chose to first convert the list into its unique elements using set(), ensuring that we don't consider duplicate values. Then, I sorted the unique numbers in descending order using the sorted() function with a custom comparator (implicitly by default sorting in Python). Since we now have distinct and sorted numbers, finding the second largest is just selecting the second to last element in the list. However, when all numbers are the same (i.e., only one unique value), I return None as per requirements. This approach ensures concise code while still handling the edge case of only one unique number.
