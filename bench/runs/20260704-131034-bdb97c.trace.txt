To solve this task, I used the built-in `sorted()` function in Python with a custom key function that considers both word length and reverse alphabetical order. By using enumerate(), I was able to count the position of each word as it is sorted. This way, the k-th word can be easily retrieved from the sorted list. To handle the edge case when k is out of range, I added a condition to check if k is within the valid range (1 <= k <= len(sorted_words)). If not, the function returns None.
