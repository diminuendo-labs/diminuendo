I wrote a generator function that checks each element of the input list. If an element is a number or string, it's yielded as-is. Otherwise, the function calls itself recursively to handle nested lists within the current one. I chose this approach because it applies depth-first search and handles lists within lists at any level, while ensuring that the flattening only occurs up to one level. An edge case handled here is when the input list contains deeply-nested structures (such as a list containing another list, which contains another list, etc.), where the function still works without infinite recursion.
