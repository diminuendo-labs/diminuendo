The function `nth_page` takes three arguments: a list of `items`, the desired `size` of each page, and the page number `n`. It first checks if `size` is zero or negative, or if `n` is less than 1. If any of these conditions are met, it returns an empty list as specified in the criteria.

Next, it calculates the starting index (`start`) and ending index (`end`) for the desired page using the formula `(n - 1) * size`. It then checks if the `start` index is beyond the length of the list or if the `end` index is less than or equal to zero. If either condition is true, it returns an empty list as this indicates that the requested page does not exist.

If all conditions are met, it slices the list from `start` to `end` and returns the resulting sublist, which represents the desired page. This approach ensures that the function handles edge cases correctly while providing a concise and readable implementation.
