I first created an empty list to store the runs of consecutive integers, and initialized `start` and `end` as `None`. I then iterated through a set containing distinct numbers from the input list, sorted in ascending order. When encountering a new number for the first time, it becomes the start of a run. If the next number is one greater than the current `end`, it increments `end`. Otherwise, if there's a gap, I append the previous run as a range to the list and reset the start and end for the next run. After iterating through all the numbers, I join the runs with commas and return the result. The edge case handled here is when an empty list is provided. In that case, the function returns an empty string.
