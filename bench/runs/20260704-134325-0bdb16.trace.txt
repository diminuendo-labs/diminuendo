3. **Merging Logic:** A single loop iterates through the sorted pairs starting from the second element. For each pair, it checks if there is an overlap or if the current pair is within distance 1 of the last merged pair. If so, it extends the end of the last merged pair to cover the current one. Otherwise, it appends the current pair as a new segment in the merged list.

This approach ensures that all overlapping and close pairs are merged correctly while maintaining the sorted order of the ranges in the output.
