I approached the problem by first initializing a list `runs` containing the first number in the input list and its value. Then I iterate through the remaining numbers in the list, incrementally updating the end value of the current run if the new number is one greater than the end value of the current run, or creating a new run for numbers that don't follow this rule.
Finally, I convert each run into a string, with single numbers staying as single numbers and ranges being converted into 'start-end' format. The resulting list of strings is then joined with commas to create the final output. One edge case I handled is when the input list is empty, in which case an empty string is returned.
