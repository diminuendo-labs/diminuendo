In this implementation, I first remove duplicates from the input list by converting it into a set. This approach ensures that there are no duplicate values in the set and makes further operations more efficient as searching for an item in a set is faster than in a list. To find the second largest value, I sort the elements of the set and extract the last two elements, which correspond to the maximum and the second-maximum values respectively. If the second-largest element is equal to the largest one (i.e., there are only two distinct values), then I return None to indicate that there's no valid second-largest value in this case.
