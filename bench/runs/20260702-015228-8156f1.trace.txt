I used a simple counter to keep track of the balance between opening and closing parentheses. As I iterate through the string, I increment the counter for each '(' and decrement it for each ')'. If at any point the counter becomes negative, it means there's a closing parenthesis before its corresponding opening one, so the function returns False. Finally, if the counter is zero after processing all characters, the parentheses are balanced, and the function returns True. This approach handles the edge case of an empty string by returning True, as no parentheses can be unbalanced in an empty string.
