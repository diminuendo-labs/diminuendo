I implemented this function by first checking for an empty input list, which is a crucial edge case as it returns an empty list immediately. Next, I sorted the input pairs based on their start values, which makes it easier to compare and merge adjacent ranges. The logic of merging overlapping or near ranges was then implemented in a loop where each range is compared with its predecessor; if they overlap or are within distance `d`, they are merged by updating the end value of the last range. If not, the current range is added to the result list. This approach ensures that ranges are processed in order and minimizes unnecessary comparisons.
