I implemented the function by first sorting the input list using a custom sorting key that prioritizes length and then breaks ties alphabetically. I used Python's built-in `sorted` function with a lambda function as the sort key, which returns a tuple of two values: the length of each word (which is the primary sorting criterion), and the reverse of the word itself (to break ties). This approach has a time complexity of O(n log n) due to the sorting. The edge case I considered is when k exceeds the number of words in the list, in which case an `IndexError` exception is raised, and the function catches this exception to return None instead.
