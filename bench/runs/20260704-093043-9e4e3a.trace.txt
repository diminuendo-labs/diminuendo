In this solution, I used a stack data structure to keep track of opening parentheses as they appear in the input string. Whenever a closing parenthesis is encountered, if the stack is empty or the top element of the stack isn't an opening parenthesis, the function returns False. If the loop finishes and the stack is still non-empty, it means that some opening parentheses were not properly closed, so the function also returns False. If the stack is empty at the end, it means all parentheses are balanced, and the function returns True. The edge case I handled here is when there is an unmatched number of closing parentheses in the input string before encountering any opening parentheses.
