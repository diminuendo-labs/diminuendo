I approached this task by iterating through each character in the input string. Whenever I encounter an opening square bracket, I increment a depth counter and compare it to the maximum depth found so far. If I encounter a closing square bracket, I first check if there is an active depth before decrementing the depth counter. In case of a closing bracket without its corresponding opener or an opener without a corresponding closing bracket, I return -1. Additionally, I handle the edge case where the input string is empty by setting the maximum depth to 0 and returning it if no characters were processed.
