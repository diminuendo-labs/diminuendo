First, I ensured that the list is sorted and distinct by using the built-in `sorted()` function with a set to avoid duplicates. Then, I created an empty list called `runs` which will store the ranges found in the list. Initially, I set the start and end variables to the first number in the list. After that, I iterated through the sorted list and compared each number to the current start and end values. If the current number is one greater than the current end, it's part of the same range, so I updated the end variable. Otherwise, if there's a gap between the current number and the end, I added the current run (if any) to the `runs` list as a formatted string, reset the start to the current number, and continued with the next number in the list. Finally, after iterating through all numbers, I add the last number if it's not already included in the runs list (since the last iteration won't update the end variable). The function returns a comma-separated string of the ranges found in the input list or just the single value if there are no ranges. An edge case handled here is when the list contains only one number, as it will still return that number instead of an empty string.
