In this approach, I first check if the input list is empty and return an empty string in that case. Then I iterate through the list, keeping track of a starting number `start`. If the current number is one more than the current start, it means we're within a run, so I skip that number. If the current number is greater than the current end plus one, it means we've ended a run, and if the start isn't equal to the end (meaning there are duplicates), I append the range from start to end into the result list. In all other cases, I simply append the current number as-is into the result list. At the end of the loop, I handle the last range or single number in a similar fashion. Finally, I join the result list with commas and return the string. The edge case handled here is when there are duplicates within a run (e.g., [1, 1, 2] becomes '1-2').
