I created a dictionary `num_counts` to keep track of the occurrences of each number encountered so far. For each number in the input list `nums`, I calculated its complement (i.e., the value needed to reach the target when added to the current number). If this complement had been seen before, it means there are pairs that add up to the target, and I incremented the pair count accordingly. After checking for complements, I updated the dictionary with the current number's count. This approach ensures that each pair is counted only once without needing nested loops, making the solution efficient. The edge case of an empty list was handled by returning 0 immediately.
