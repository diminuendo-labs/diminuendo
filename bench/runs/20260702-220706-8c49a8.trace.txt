I approached this problem by first checking for the edge case of an empty list. If the input list is empty, I immediately return 0 as there are no valid pairs. Then, I used a set to store unique numbers encountered so far (`num_set`), and another set to store unique pairs that sum to the target (`pair_count`). For each number in the input list, I calculated its complement (the value needed to reach the target) and checked if it's already in `num_set`. If it is, I add a sorted version of the pair to `pair_count` to ensure distinct unordered pairs are counted correctly. Finally, I return the size of `pair_count`, which represents the number of distinct unordered pairs with the given target sum.
