I used a simple stack-based approach to solve this problem. The idea is to push opening parentheses onto the stack and pop them off whenever we encounter a closing parenthesis. If at any point we try to pop an empty stack (meaning there's no matching opening parenthesis for the current closing one), we immediately return False. Finally, after processing the entire string, if the stack is empty it means all parentheses were correctly matched, so we return True; otherwise, we return False. This approach handles the edge case of an empty string (which is balanced) implicitly, as there are no parentheses to process in this case.
