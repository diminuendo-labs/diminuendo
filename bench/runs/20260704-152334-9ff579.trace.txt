I approached this problem by first splitting the input string into individual tokens using the underscore delimiter. Then, I defined a helper function `camel_case_token` that determines whether to apply camel case conversion or leave the token unchanged (for exception cases). The main function uses list comprehension to apply `camel_case_token` to each token and join the results with no separator. This solution handles the edge case where 'id' appears as the second token, converting it to lowercase if it is not in the exceptions list.
