I approached this problem by iterating through the input list and maintaining two variables, 'last' to store the last seen number and 'count' to keep track of its occurrences. When we encounter a new number, we check if it's the same as 'last'. If yes, increment 'count', else append the range found so far (from 'last' with the count) to our result list and start tracking the new number. Finally, join the results into a string with commas in between and return. The edge case I have handled is when the input list is empty; it returns an empty string instead of throwing an error.
