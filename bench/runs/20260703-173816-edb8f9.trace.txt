To solve this problem, I used the property that a rotation of string `a` has the same length as `a`, and when concatenated with itself, it forms twice the length of the original string. This means if `a + a == b * 2`, then `b` is a rotation of `a`. To handle an edge case where one string may be a proper rotation (not full rotation) of another, I used the check `len(a) != len(b)` to ensure both strings have the same length when they're valid rotations.
