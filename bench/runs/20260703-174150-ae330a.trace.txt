The approach for this function involves using a stack data structure to keep track of opening parentheses as they appear in the string. Each time a closing parenthesis is encountered, we check whether there's an opening parenthesis at the top of the stack or if the stack is empty. If either condition is true, the function immediately returns False, indicating that the parentheses are not balanced. If the function iterates through the entire string without finding any unmatched closing parentheses, it means the parentheses are balanced and the function returns True.

To handle an edge case, I've accounted for a scenario where there are no parentheses in the input string at all (an empty string). In this case, the function will correctly return True as an empty string is considered to have balanced parentheses.
