I sorted the input list of ranges first, then iterated through them. If the end of the current merged range was greater than or equal to the start of the next pair, I updated the end of the current merged range. Otherwise, I added the next pair to the list of merged ranges. An edge case this approach handles is when there are no overlapping ranges (e.g., [1, 3] and [7, 9]), as it will correctly return a list with each original range.
