The function `max_index` iterates through the list in reverse order, finding the index of the maximum value. This ensures that when multiple maximum values exist, it returns the last occurrence, not the first. If no elements are present in the list, it returns `None`. To achieve conciseness and readability, I used slicing (`[::-1]`) to reverse the list before searching for the index of the maximum value. The edge case handled here is when the input list is empty.
