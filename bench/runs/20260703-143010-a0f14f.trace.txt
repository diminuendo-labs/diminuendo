I chose to use a set data structure to keep track of the numbers encountered so far, which allows for efficient lookups and insertions. This approach has a time complexity of O(n), where n is the length of the input list. To ensure that each distinct pair is counted only once, I used another set to store the pairs, with each pair represented as a sorted tuple. The edge case where an empty list is passed in is handled explicitly by returning 0 if the input list is empty.
