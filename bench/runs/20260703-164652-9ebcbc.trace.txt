I used a simple approach by iterating through the input string `s`. For each character, I check if it matches the previous character and if so, calculate the length of the current run. If this length is greater than the maximum length found so far, I update the max_length and max_char variables accordingly.

In case there are multiple runs with the same maximum length, I store the last such occurrence by using the prev_char variable to keep track of the previous character. If the input string is empty, I return an empty string as specified in the task's criteria. The edge case I considered here was handling multiple consecutive runs of equal lengths.
