To solve this problem, I first removed duplicate characters from the input string `s` using a set. Then I iterated over the remaining unique characters and calculated the length of each consecutive run in the string. To avoid counting multiple instances of the same character, I only considered runs that were not followed by the same character. The maximum length found was stored along with its corresponding character. In case of a tie for the longest run, I ensured to return the character from the last such run instead of the first one. The function returns an empty string if no characters appear more than once in the input string. An edge case handled here is when there are no consecutive runs of any character (e.g., "ABC").
