I approached this problem by using the built-in `max` function with a custom key. The key is a lambda function that returns a tuple containing the value and the negative of its index. This way, when there are multiple occurrences of the maximum value, Python will compare their indices as strings, which results in the last occurrence coming first due to lexicographic ordering. I also handled the edge case of an empty list by returning `None` directly at the beginning of the function.
