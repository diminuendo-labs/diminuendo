I defined the function `chunk` that takes two parameters: a list (`lst`) and an integer (`size`). The first edge case I considered was when `size` is less than or equal to zero. In this case, I immediately return an empty list because we cannot split the list into chunks of non-positive size. For valid sizes, I used a list comprehension to create chunks by slicing the original list from `i` to `i + size` in each iteration, where `i` ranges from 0 to the length of the list with a step of `size`. This approach ensures that the function is both concise and readable while correctly handling the edge case where the input size is non-positive.
