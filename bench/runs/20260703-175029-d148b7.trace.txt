I approached this problem by iterating through the input list and keeping track of the current start and end values of each run. Once a new run starts (when the next number is greater than the current end + 1), I append the previous run to the result, reset the current start and end to the new number, and continue with the next iteration. After the last number, I also append the final run to the result. To ensure distinct values, each value in the result is either a single number (when it starts and ends the same range) or a range of numbers between the start and end points (when there are multiple consecutive instances of a number). If the input list is empty, I return an empty string.
