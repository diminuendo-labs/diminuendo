I started by sorting the list of pairs to ensure that any overlapping or adjacent ranges are next to each other. This simplifies the merging process. I then iterated through the sorted list, keeping track of the last merged range and comparing it with the current one. If there was an overlap or adjacency (i.e., the start of the current range is less than or equal to the end of the last merged range), I merged them by updating the end of the last merged range to be the maximum of its current end and the end of the current range. If they did not overlap, I simply appended the current range to the result list. This approach ensures that all overlapping ranges are merged into a single range, and the final list is sorted by start.
