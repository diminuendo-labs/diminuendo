I approached this problem by first handling the edge case of an empty list, returning an empty string as required. Then I used a two-pass approach to group consecutive numbers together: on the first pass, I iterated through the sorted set of numbers and checked for consecutive values; if they were not consecutive, or if it was the first number in a range, I appended a new range to my result list. On the second pass, I iterated through the original list again to handle the case where a single number is present (i.e., no ranges should be created). This approach allowed me to keep the code concise and readable while still handling all edge cases.
