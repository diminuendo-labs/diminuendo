This solution uses the built-in functions `max()` and `index()`. To handle the edge case where the maximum value is not present, we first reverse the input list using slicing (`[::-1]`), then check if the maximum value exists in the reversed list. If it does, we get its index from the reversed list to ensure the last occurrence; otherwise, we get the index from the original list. This approach ensures that the function returns the index of the last occurrence when multiple maxima are present and handles edge cases where the maximum value is not in the list.
