I created a function `merge_within` that takes a list of pairs as input, sorts them first to make the process more straightforward, and then iterates through each pair. If the start of the current pair is less than or equal to the end of the last merged range, I merge the two ranges by updating the end of the last merged range to be the maximum between its original end and the end of the current pair. If the current pair's start is greater than the end of the last merged range, I append the current pair to the result list to ensure that separate ranges remain separate. An edge case handled here is when there are no pairs (an empty list), in which case an empty list is returned.
