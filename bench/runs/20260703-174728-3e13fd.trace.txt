The approach here is to first convert the list of numbers into a set, which allows for constant time lookup. We iterate over the original list (to avoid sorting) and for each number, we check if its complement exists in the set (the difference between target and current number), but only if it's different from the current number itself to handle the edge case where a number equals the target multiple times but should still count as distinct pairs. If a match is found, we increment the counter by 1. The function returns the final count of pairs.
