I used two lists to handle the runs of consecutive numbers and store them appropriately. I initialized a current number variable to keep track of the last processed number. If a new number is encountered that is one more than the current number, I add it to the last run instead of starting a new one. If the current number is not followed by another number within the loop (indicating a single number), it remains as a single number in the result string. I use list comprehension to create the final result string, using 'start-end' format for runs and simply the start value for single numbers. The edge case that this function handles is an empty list which returns an empty string.
