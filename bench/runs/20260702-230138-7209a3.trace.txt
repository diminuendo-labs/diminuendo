I solved the problem by first removing duplicates and sorting the input list, then iterating over it to identify consecutive runs. I used a `current_range` variable to keep track of the current range, and added elements to this range as long as they were consecutive. Once a non-consecutive number was encountered, I converted the current range to a string (either as 'start-end' or just 'start') and added it to the list of ranges. Finally, I joined all the ranges together with commas and returned the result. The edge case for an empty input list is explicitly handled by returning an empty string immediately at the start of the function.
