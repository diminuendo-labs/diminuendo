I chose to iterate through the sorted list of numbers and keep track of the current range using two variables, `start` and `end`. When a different number is encountered, I check if the start equals the end. If they are equal, it means there was only one number in that range, so I append it as a single number to the result. If they are not equal, it means there was more than one number, so I append the range in 'start-end' format. This approach ensures that consecutive runs are collapsed into ranges and handles edge cases like an empty list or a list with all unique numbers.
