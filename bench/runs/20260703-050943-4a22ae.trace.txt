Then, I iterated through the sorted list, identifying consecutive sequences by checking if each number is exactly one more than the previous. If a sequence of numbers is found, it's collapsed into a 'start-end' format; otherwise, individual numbers are added to the result list. Finally, the results are joined with commas and returned as a single string. This approach ensures that the function handles both consecutive sequences and isolated numbers correctly while adhering to the criteria for correctness and readability.
