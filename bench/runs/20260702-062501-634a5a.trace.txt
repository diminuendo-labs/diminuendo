I chose to use a stack data structure to keep track of the opening parentheses encountered so far. As I iterate through the string from left to right, whenever I encounter an opening parenthesis, it gets pushed onto the stack. When I encounter a closing parenthesis, I check if the stack is empty (which would mean there's no matching opening partner). If the stack is not empty, I pop the top element off the stack (the matching opening partner). Finally, after iterating through the entire string, I return whether the stack is still empty (meaning all parentheses were balanced) or not. This approach handles the edge case where an empty string is provided as input.
