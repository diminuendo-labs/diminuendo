I chose to iterate through the string and use a counter to track the current depth of square brackets. When an opening bracket `[` is encountered, the counter is incremented, and the maximum depth seen so far is updated if the current depth exceeds it. For each closing bracket `]`, the counter is decremented. If at any point the counter becomes negative, indicating a mismatched bracket, the function returns -1. After processing all characters, if the counter is not zero, meaning there are unmatched opening brackets, the function also returns -1. Finally, if no mismatches were found and the maximum depth was greater than 0, the function returns the maximum depth; otherwise, it returns 0 for an empty string or a string without any brackets.
