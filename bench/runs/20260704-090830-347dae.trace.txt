I used a combination of sorting and iterating through the list to merge overlapping ranges. I first sorted the input list by the start value using a lambda function. Then, I started by adding the first range to a new list representing the merged ranges. For each subsequent range, if its start was greater than or equal to the end of the last merged range, I updated the end of the last merged range; otherwise, I added the current range to the merged ranges list. This handles the edge case where there are multiple overlapping ranges that need to be merged into a single one, like in the example [1, 3] and [4, 6].
