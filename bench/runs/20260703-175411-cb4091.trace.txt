In this solution, I used a single pass through the list to generate the runs of consecutive numbers and then join them with commas. The edge case handled here is when a number appears more than once in the input list. Instead of listing each duplicate separately, they are grouped together using the range syntax (e.g., '2-2' for two 2's). If a single number does not have any duplicates before or after it, it remains as a single number rather than a range (e.g., '1', '3', and '5').
