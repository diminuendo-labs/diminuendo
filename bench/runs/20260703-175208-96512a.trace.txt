I approached this task by first sorting the input list and converting it to a set to ensure that each distinct value is counted only once. Then, I iterated through the sorted list, keeping track of ranges in a list called `runs`. If a number was consecutive with the previous one, I skipped adding it to the runs. However, if it was a duplicate or not consecutive, I either updated the end of the current range or added the new number as a single run. Finally, I joined the runs together using commas and returned the final string. The edge case handled is when the input list is empty, in which case an empty string is returned instead of raising an error.
