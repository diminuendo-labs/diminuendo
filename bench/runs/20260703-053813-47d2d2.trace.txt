To solve this problem, I chose to split the string into words using Python's `split()` method, which handles multiple consecutive whitespace characters and also ignores leading/trailing spaces. This ensures that the function works correctly even for strings with irregular spacing. The shortest word is then found by accessing the last element of the list (since ties should return the last word). If the input string is empty or contains only whitespace, an empty string is returned as specified in the edge case handling.
