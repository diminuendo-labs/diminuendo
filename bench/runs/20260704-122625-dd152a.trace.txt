I approached this task by first sorting the input list based on the start points of the ranges. Then, I iterated through the sorted list and compared each range with the last merged range. If the new range starts after the end point of the last merged range, it was appended to the merged_list. Otherwise, if the new range overlaps with the last merged range or shares an endpoint, I updated the end point of the latest merged range. This approach ensures that the output is sorted by start and handles the edge case where the starting point of a new range is less than the end point of the last merged range.
