I implemented an iterative generator expression that checks each element of the input list. If the element is a primitive data type (integer, float, or string), it's added directly to the result. Otherwise, if the element is a list, it will be flattened recursively by calling the `flatten_once` function on the nested list. This approach ensures that exactly one level of nesting gets flattened, while preserving deeper levels of nesting. The edge case I handled is when an item in the input list is already a flat list or a single primitive data type, as it will remain unchanged.
